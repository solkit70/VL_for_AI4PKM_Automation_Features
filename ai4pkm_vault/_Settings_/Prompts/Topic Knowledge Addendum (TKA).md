Update topic notes using daily roundup content to add relevant summaries and maintain knowledge connections.

## Input
- [[Generate Daily Roundup (GDR)]] output file
- Existing topic notes in Topics directory
- Content from daily roundup requiring categorization

## Output
- Updated topic notes with new summaries
- Wiki links to source materials
- Generalizable lessons and experiences
- Pointer-style entries (one line per source)

## Main Process
```
1. TOPIC IDENTIFICATION
   - List relevant topics from roundup content
   - Identify existing topic notes to update
   - Map content to appropriate topic categories

2. CONTENT PROCESSING
   - For each content item in GDR
   - Extract generalizable lessons and experiences
   - Create concise summary for topic note

3. TOPIC NOTE UPDATES
   - Add relevant summaries to topic notes
   - Include wiki links to source materials
   - Maintain one line per source format
```

## Caveats
### Content Standards
⚠️ **CRITICAL**: Focus on generalizable lessons and experience

### Formatting Requirements
- One line per source for consistency
- Quote sources using wiki link format
- Don't add excessive details from each note

### Linking Standards
- Make sure to quote sources in wiki link form
- Goal is to provide pointers to each note
- Maintain clear connection between topic and source
